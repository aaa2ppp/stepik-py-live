Эта форма создает новую случайную жизнь на поле заданного размера. Допустимые значения размеров от 1 до 200*.

Остальные поля устанавливают начальные параметры запроса /live?<query>:

serial=<int>
номер поколения начиная с 0 (если выбрать большое значение совместно с большим размером поля игры, расчет истории может занять длительное время);
autoupdate={on|off}
включить авто обновление, по умолчанию off;
update_period=<int>
период обновления в ms, минимальное допустимое значение 100 (при малых значениях, для большого поля и/или слабом процессоре, будут просадки);
js={on|off}
включает/отключает использование JavaScript на стороне клиента, по умолчанию on (если выключить, то авто обновление работать не будет).

Вы можете изменять их в адресной строке в любое время, после создания жизни. Не забывайте нажать Enter, что бы применить. Кнопка Остановить/Продолжить управляет параметром autoupdate. Для остальных параметров элементов управления нет**.

* В настоящий момент, если используется оптимизация расчета нового поколения (поведение по умолчанию), ширина поля выравнивается до числа кратного 16 в большую строну. Вы можете запустить приложение с установленной переменной окружения NAIVE_ALGO=1, чтобы отключить оптимизацию.

** Если включено авто обновление, что бы актуализировать параметр serial, его нужно остановить. Иначе, при обновлении страницы, вы откатитесь на начало игры или момент последней остановки.
